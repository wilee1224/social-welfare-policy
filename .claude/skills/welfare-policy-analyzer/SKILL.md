# 사회복지 정책 분석 스킬

이 스킬은 원문 데이터에서 의미 있는 정책 변경사항을 추출하고 구조화하는 방법을 정의한다.

## 분석 단계

### Step 1: 정책 변경 여부 판단

원문에서 다음 신호어를 탐지하여 정책 변경 여부를 판단:
- 인상/인하, 확대/축소, 완화/강화
- 신설/폐지/개편/변경
- ~부터 적용, 시행, 개정

단순 안내, 행사, 채용은 제외한다.

### Step 2: 이전 정책 파악

기존 정책 내용을 파악하는 우선순위:
1. `data/policies.json`에 동일 카테고리 기존 항목이 있으면 참조
2. 보도자료 원문에 "기존 ~에서 ~로 변경" 형태로 명시된 경우 원문 인용
3. 위 두 방법이 모두 불가하면 WebSearch로 이전 제도 배경 확인 (출처 명시 필수)
4. 확인 불가 시 "기존 정책 확인 불가"로 표시

### Step 3: 핵심 변경사항 추출

핵심 변경사항은 3~5개로 정리:
- 수치가 있으면 반드시 포함 (예: "150만→200만 원")
- 대상 범위 변화 포함 (예: "65세 이상→60세 이상으로 확대")
- 절차 변화 포함 (예: "방문 신청 → 온라인 신청 추가")

### Step 4: 카테고리 분류

| 키워드 | 카테고리 |
|--------|---------|
| 의료급여, 부양의무자, 진료비 | 의료급여 |
| 기초생계, 생계급여, 주거급여 | 기초생활보장 |
| 육아휴직, 아동수당, 보육, 출산 | 육아지원 |
| 장기요양, 기초연금, 노인돌봄 | 노인복지 |
| 장애연금, 활동지원, 발달재활 | 장애인복지 |
| 공공임대, 전세, 주거급여 | 주거복지 |
| 실업급여, 고용장려, 취업 | 고용지원 |

## 품질 기준

- **원문 이외 내용 절대 추가 금지** — 창작이나 추측은 팩트체커의 적발 대상
- 수치는 원문 그대로 (반올림, 환산 금지)
- 하나의 보도자료에 여러 정책이 있으면 각각 별도 항목으로 분리
